# Contributing

Thank you for helping build Fabric as Community.

Contribution is not limited to code. You can contribute through documentation, ideas, research, testing, issue triage, review, design, community support, and partner feedback.

## Ways to contribute

- open an issue
- improve documentation
- submit a proposal
- review a pull request
- test an idea
- write examples
- improve governance language
- report security concerns responsibly
- help another contributor

## Contribution flow

```text
Find or open issue -> Discuss -> Submit change -> Review -> Decision -> Merge or close -> Record evidence
```

## Contributor expectations

Contributors should:

- be respectful
- explain context
- keep changes focused
- link evidence
- accept review with care
- disclose conflicts of interest where relevant
- avoid hidden or private decision-making for public matters

## Pull request checklist

Before opening a pull request:

- [ ] The change has a clear purpose.
- [ ] The change is scoped and reviewable.
- [ ] Related issues or proposals are linked.
- [ ] Governance or trust impact is considered.
- [ ] Security impact is considered.
- [ ] Documentation is updated when needed.

## Proposals

Use proposals for changes that affect governance, roles, partnerships, architecture, trust, or the community operating model.

## Respect

Fabric is a community graph. Every contribution affects people.

Be clear, kind, and accountable.
