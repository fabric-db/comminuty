# Governance

Fabric governance is community governance expressed as a transparent graph.

Governance is not authority by title. Governance is responsibility proven through contribution, review, decision quality, and care for the community.

## Principles

1. **Open participation** — anyone may propose, discuss, document, or contribute.
2. **Earned responsibility** — privileges come after visible contribution.
3. **Evidence-based decisions** — decisions must link to context, discussion, and impact.
4. **Human accountability** — automation may assist, but people remain responsible.
5. **Deny capture** — no single vendor, funder, or maintainer should control the community alone.
6. **Reversibility** — prefer changes that can be safely rolled back.
7. **Transparency** — decisions are recorded in public unless security or privacy requires limits.

## Roles

| Role | Responsibility | Privilege |
| --- | --- | --- |
| Community Member | Participate respectfully | Open issues, discussions, proposals |
| Contributor | Submit work | Pull requests, docs, proposals |
| Reviewer | Review work | Review recommendations |
| Maintainer | Steward project areas | Merge, release, moderate |
| Governance Maintainer | Steward process | Approve governance changes |
| Partner | Support adoption and delivery | Partner proposals and public commitments |

## Decision types

| Type | Examples | Approval |
| --- | --- | --- |
| Routine | docs, small fixes, typo, links | maintainer review |
| Technical | architecture, schema, runtime, API | proposal + maintainer review |
| Governance | roles, voting, conduct, trust model | governance proposal |
| Security | vulnerability handling, access changes | security maintainers |
| Partnership | ecosystem commitments | partner proposal + maintainer review |

## Decision states

```text
proposed -> discussed -> accepted | rejected | deferred | escalated -> recorded
```

## Maintainer expectations

Maintainers must:

- act in the interest of the community
- document decisions
- avoid conflicts of interest
- review respectfully
- protect contributors from abuse
- prefer open process
- step back when they cannot be fair

## Conflict handling

Conflicts should be handled with care, privacy when needed, and clear outcomes.

Escalation path:

```text
direct discussion -> maintainer mediation -> governance maintainer review -> recorded outcome
```

## Changing governance

Governance changes require:

1. a proposal
2. public discussion window
3. maintainer review
4. recorded decision
5. update to this document
